Database Components

S.No Component Definition
1 Tables It is collection of data elements organized in shape of rows and columns.
2 Data Type Data type is a data storage format that can contain a specific type or range of values.
3 Field It is the smallest component in a database. It is where the actual data stored during the data entry.
4 Views In a database, data resides in tables, but views provide a virtual representation without storing data themselves.
5 Record Record is a single entry of data in a table, also refers as tuples or rows. Record is made up of two or several data items.

Database Data Types

S.No DataType Description
1 Integer Holds only whole numbers. Example: 300, 50
2 FLoating Point Numbers with decimal point. Example: 94.44, 6.75
3 Characters Stores only one character. Example: A, B, C
4 String Stores a combination of letters, numbers and special characters. Example: XYZ-123
5 Boolean Stores only boolean values. Example: 0, 1
5 Data And Time Stores date and time in specified format. Example: 2007-02-28, 12:45:12.23456

DBMS VS Flat File System

DBMS
  1. Multiple users can access simultaneously
  2. Allow non-duplicate and integrity
  3. Capable of handling huge sets of data
  4. Good for small, medium and large business
FFS
  1. Only one user can access at a time
  2. Increases duplicate and redundant data
  3. Suitable only for small sets of data
  4. limited only to the smallest data management needs
